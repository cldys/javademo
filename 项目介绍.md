# 项目介绍

这是一个基于 Spring Boot 4.0.4 和 Java 17 的学习型 Web 项目，当前已经整理成更接近正式业务的分层结构。

## 目标

这个项目用一个最小的 `User` 示例，演示后端项目里最常见的分层方式：

- `Controller`：接收 HTTP 请求，返回 JSON 响应
- `Service`：组织业务流程，处理用例
- `Domain`：表达业务对象本身
- `DTO`：承载请求和响应数据
- `Exception`：处理业务异常

## 当前结构

```text
src/main/java/com/example/javademo
├── JavademoApplication.java
└── user
    ├── controller
    │   └── UserController.java
    ├── service
    │   └── UserService.java
    ├── domain
    │   └── User.java
    ├── dto
    │   ├── UserCreateRequest.java
    │   └── UserResponse.java
    └── exception
        └── UserNotFoundException.java
```

## 接口

当前提供 3 个接口：

- `POST /users`：创建用户
- `GET /users/{id}`：查询用户
- `POST /users/{id}/birthday`：让用户年龄加 1

## 示例请求

创建用户：

```bash
curl -X POST http://localhost:8080/users \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice","age":17}'
```

查询用户：

```bash
curl http://localhost:8080/users/1
```

过生日：

```bash
curl -X POST http://localhost:8080/users/1/birthday
```

## 代码阅读顺序

如果你想学习整个调用链，建议按这个顺序看：

1. `UserController`
2. `UserService`
3. `User`
4. `UserCreateRequest`
5. `UserResponse`

## 设计说明

- `UserController` 只处理请求和响应，不写业务判断。
- `UserService` 负责创建、查询、更新用户。
- `User` 负责自己的状态和行为。
- `UserCreateRequest` 是输入 DTO，带校验。
- `UserResponse` 是输出 DTO。
- `UserNotFoundException` 用来把“找不到用户”映射成 404。

## 备注

当前 `UserService` 用的是内存 Map，方便学习调用链和分层，不依赖数据库。后续如果要做正式业务，可以再替换成 `Repository` 和数据库持久化。
